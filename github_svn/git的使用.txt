git配置:（公司git不用配置的，配置自己所用的github账号就行）
显示当前的Git配置：git config --list
设置提交代码时的用户信息：
git config user.name "githud用户名"
git config user.email "邮箱地址"


git使用:

git下载：git clone git链接 
路径指向项目：cd 项目名

查看本地和远程所有分支：git branch -a
切换分支：git checkout master----------重点：当你发现切换不了时：用git fetch------更新出远程分支
切换并创建分支：git checkout -b 新分支名


提交dev分支：
查看状态：git status -s
拉取更新分支：git pull origin 分支名----------重点（本地更新远程）
选择所有文件进入暂存区：git add .
进入版本库：git commit -m '提交说明'
提交分支：git push origin 分支名


在master里合并dev:
git checkout dev
git pull origin dev
git checkout master
git pull origin master
合并分支：git merge dev
情景1.合顺利：git push origin master
情景2.合冲突：出现（master|MERGING）
git status -s   手动修改红色文件
git add .
git commit -m '提交说明'       此时（master|MERGING）消失
git push origin master


合并冲突产生原因：同时修改同个文件

用git merge dev会遇到的问题：
Please enter a commit message to explain why this merge is necessary,
https://www.cnblogs.com/wei325/p/5278922.html
E45: 'readonly' option is set (add ! to override)
https://baijiahao.baidu.com/s?id=1642180606627332320&wfr=spider&for=pc



用git pull origin dev遇到冲突时：
先保存一份代码
清空修改代码：git stash
git pull origin dev
再添加本地的修改

返回历史记录：
查看提交历史日志：git log
返回版本：git reset --hard 139dcfaa558e3276b30b6b2e5cbbb9c00bbdca96

选择历史版本，合并进当前分支：
git log
git cherry-pick 1bb8e0275fce3fddaa4b62511735ae3b35472e82




取消已经add的文件的备份：git reset

显示被修改文件的修改统计信息，添加或删除了多少行：git log --stat

当前的路径：pwd

初始化：git init

取消合并： git merge --abort

展示历史
git log
git show  1bb8e0275fce3fddaa4b62511735ae3b35472e82
